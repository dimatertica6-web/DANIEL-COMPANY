Политика конфиденциальности

RfiMain · OSINT Framework
Дата вступления в силу: 14 июля 2026 г.

1. Общие положения

Настоящая Политика конфиденциальности регулирует сбор, обработку и хранение персональных данных пользователей при использовании OSINT-фреймворка RfiMain (далее — «Сервис»).

Используя Сервис, вы соглашаетесь с условиями настоящей Политики. Если вы не согласны с условиями, пожалуйста, прекратите использование Сервиса.

2. Какие данные мы собираем

  • Данные пользователя Telegram: ID, username, имя, фамилия (предоставляются Telegram при авторизации).
  • История запросов: тип запроса, данные запроса, результат, время выполнения, источник.
  • Цели и отчеты: сохраненные пользователем цели, созданные отчеты.
  • Технические данные: IP-адрес, время доступа, тип устройства, версия браузера.
  • Данные кэша: результаты API-запросов для ускорения работы (хранятся ограниченное время).

3. Как мы используем данные

  • Обеспечение работы Сервиса (обработка запросов, кэширование).
  • Улучшение качества работы (аналитика, статистика).
  • Предоставление персонализированных функций (сохранение целей, отчетов).
  • Обеспечение безопасности (защита от злоупотреблений, спама).
  • Уведомления о статусе задач и обновлениях Сервиса.

4. Хранение данных

Данные хранятся в защищенной SQLite-базе данных на сервере. Доступ к базе данных ограничен и осуществляется только через авторизованные сервисы.

  • История запросов хранится бессрочно для обеспечения работы Сервиса.
  • Кэш API-запросов хранится ограниченное время (по умолчанию 1–24 часа).
  • Данные пользователя хранятся до момента удаления аккаунта.

5. Передача данных третьим лицам

Сервис использует внешние API для выполнения запросов (AbuseIPDB, VirusTotal, ipinfo, WhoisXML, Etherscan и др.). При выполнении запросов передаются следующие данные:

  • IP-адрес, домен, email, телефон, URL, хеш, адрес кошелька.
  • Результаты запросов могут передаваться в кэш для оптимизации.

Сервис не передает персональные данные пользователей (ID, username) третьим лицам, за исключением случаев, предусмотренных законодательством.

6. Безопасность

Мы принимаем следующие меры для защиты данных:

  • Шифрование соединений через HTTPS.
  • Использование прокси-серверов для анонимизации запросов.
  • Ограничение доступа к базе данных.
  • Регулярное обновление зависимостей и компонентов.

Однако, никакой метод передачи данных через интернет не является абсолютно безопасным, и мы не можем гарантировать 100% защиту.

7. Права пользователей

  • Доступ: вы можете получить доступ к своим данным через команды бота (/stats, /targets, /report).
  • Изменение: вы можете обновить свои данные через Telegram.
  • Удаление: вы можете запросить удаление своих данных, обратившись к администратору.
  • Отказ от уведомлений: вы можете отключить уведомления через настройки бота.

8. Cookies и трекеры

Сервис не использует cookies и сторонние трекеры. Вся аналитика основана на внутренних данных (история запросов, события).

9. Изменения в Политике

Мы оставляем за собой право вносить изменения в настоящую Политику. Актуальная версия всегда доступна по ссылке. Изменения вступают в силу с момента публикации.
