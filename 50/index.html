Политика конфиденциальности
MelloANTI

Настоящая Политика конфиденциальности описывает, какие данные собираются, обрабатываются и хранятся при использовании Telegram бота MelloANTI, а также меры, принимаемые для их защиты.

Версия 3.0 Актуально с 28.03.2026 GDPR Compliant Полная прозрачность
Введение
MelloANTI (далее — «Бот», «Сервис») разработан для обеспечения безопасности и конфиденциальности пользователей Telegram. Настоящая Политика конфиденциальности (далее — «Политика») регулирует отношения между Администрацией Бота и Пользователем в отношении сбора, обработки, хранения и защиты персональных данных. Используя Бот, Пользователь дает свое согласие на обработку данных в соответствии с данной Политикой. Администрация Бота стремится обеспечить максимальный уровень защиты персональных данных и соблюдение всех применимых норм законодательства в области защиты данных, включая Общий регламент по защите данных (GDPR) и Федеральный закон № 152-ФЗ «О персональных данных». Бот не передает и не продает персональные данные третьим лицам, за исключением случаев, прямо предусмотренных законодательством. Все данные, передаваемые Пользователем, используются исключительно для обеспечения функциональности Бота и повышения уровня безопасности аккаунта.
Ключевой принцип: Мы собираем ровно столько данных, сколько необходимо для обеспечения безопасности и функциональности Сервиса. Никаких избыточных данных, никакой коммерциализации персональной информации.
Какие данные собираются

Обязательные данные

  • ID пользователя Telegram
  • Номер телефона (для генерации ключа)
  • Сессионные данные Telethon
  • IP-адрес при подключении
  • Отпечаток сессии (fingerprint)

Опциональные данные

  • Имя устройства (пользовательское)
  • Настройки защиты (уровень, фильтры)
  • Список заблокированных IP-адресов
  • История ключей шифрования
  • Запланированные сообщения
Важно: Бот не собирает содержимое сообщений в открытом виде. Все сообщения шифруются на стороне клиента перед отправкой. Расшифрованные тексты хранятся только локально в базе данных Пользователя и не передаются Администрации.
Использование данных

Аутентификация

Номер телефона и ID пользователя используются для идентификации и генерации уникального ключа шифрования. Без этих данных невозможно обеспечить персонализированную защиту аккаунта.

Шифрование

На основе номера телефона генерируется криптографический ключ, который используется для шифрования и дешифрования сообщений. Ключ не передается третьим лицам и хранится локально.

Защита сессий

Данные о сессиях (fingerprint, IP, устройство) используются для мониторинга активности и автоматической блокировки подозрительных подключений к аккаунту.

Безопасность

Логи действий и информация о блокировках IP используются для аудита безопасности и выявления потенциальных угроз. Данные хранятся 30 дней.

Управление

Настройки пользователя (уровень защиты, фильтры, устройства) используются для персонализации работы Бота и применения соответствующих политик безопасности.

Планирование

Данные о запланированных сообщениях используются для автоматической отправки в указанное время. Сообщения шифруются перед отправкой.

Хранение данных
Локальное хранение: Все данные Пользователя хранятся в локальной базе данных SQLite на сервере, где запущен Бот. База данных включает следующие таблицы: users, sessions, messages, logs, blocked_ips, key_rotation, contacts, scheduled_messages, broadcast_messages, filters, backups, alerts, devices. Доступ к базе данных имеет только Администрация Бота и сам Пользователь через интерфейс Бота.
Период хранения: Логи действий хранятся 30 дней. Сообщения — 90 дней. Сессии — до момента деактивации. Резервные копии — до удаления Пользователем. По истечении сроков данные автоматически удаляются.

Структура хранения

  ├── sessions — активные сессии
  ├── messages — сообщения (зашифрованные)
  ├── logs — журнал действий
  ├── blocked_ips — заблокированные IP
  ├── key_rotation — история ключей
  ├── contacts — контакты

  ├── filters — фильтры
  ├── backups — резервные копии
  └── alerts — уведомления
Меры безопасности

Шифрование данных

Все конфиденциальные данные, включая ключи шифрования и сообщения, хранятся в зашифрованном виде с использованием алгоритма AES-256. Ключи доступа к базе данных не хранятся в открытом виде.

Защита от перехвата

Все соединения между Ботом и Telegram API защищены протоколом MTProto с end-to-end шифрованием. Используются только официальные методы API.

Аудит доступа

Все действия с данными логируются с указанием времени, IP-адреса и типа операции. Это позволяет отслеживать любой несанкционированный доступ.

Автоматическая защита

Система автоматически блокирует подозрительные сессии, IP-адреса и предотвращает брутфорс-атаки. Пороги срабатывания настраиваются индивидуально.

Резервное копирование

Регулярное создание зашифрованных резервных копий данных с возможностью восстановления. Резервные копии хранятся локально.

Обновления

Бот регулярно обновляется для устранения уязвимостей и применения новых методов защиты. Все обновления проходят проверку безопасности.

Права пользователя
  • Право на доступ к своим данным
  • Право на исправление неточных данных
  • Право на удаление данных (команда /delete)
  • Право на ограничение обработки
  • Право на отзыв согласия в любой момент
  • Право на экспорт данных в JSON
  • Право на подачу жалобы в надзорные органы

Как реализовать свои права

Доступ к данным: Используйте кнопку "Профиль" или "Экспорт данных"

Удаление данных: Используйте команду /delete или очистите сессии

Отзыв согласия: Удалите Бот и очистите локальные файлы

Экспорт: Используйте кнопку "📁 Экспорт данных" для получения JSON

Жалоба: Напишите на support@melloanti.com
Использование Cookies
Бот MelloANTI не использует файлы cookies в традиционном понимании, так как функционирует через Telegram API. Однако, для обеспечения стабильной работы сессий и аутентификации, Бот использует следующие механизмы: файлы сессий Telethon (сохраняются локально в папке sessions), токены доступа (используются для авторизации в Telegram API), локальное хранилище (база данных SQLite). Все эти механизмы не являются сторонними cookie-файлами и не используются для отслеживания пользователей за пределами Бота. Файлы сессий содержат информацию об авторизации и шифровании, необходимую для поддержания активного соединения с Telegram. Они хранятся на сервере, где запущен Бот, и удаляются при завершении сессии или по команде пользователя. Пользователь может в любой момент очистить все сессии через кнопку "Очистить сессии" или удалить файлы сессий вручную.
Важно: Бот не использует сторонние аналитические сервисы, пиксели отслеживания или рекламные cookie. Единственная цель хранения данных — обеспечение безопасности и функциональности Сервиса.
Передача данных третьим лицам
Общий принцип: MelloANTI не передает и не продает персональные данные Пользователей третьим лицам. Все данные используются исключительно для обеспечения функциональности Бота и защиты аккаунта. Исключения составляют случаи, прямо предусмотренные законодательством, или когда это необходимо для выполнения требований Telegram API (например, передача номера телефона для авторизации).

Случаи передачи

  • По требованию правоохранительных органов
  • Для защиты жизни и здоровья пользователей
  • С явного согласия Пользователя
  • Telegram API (авторизация)

Администрация Бота оставляет за собой право вносить изменения в настоящую Политику конфиденциальности по мере необходимости. Все изменения вступают в силу с момента публикации обновленной версии на данной странице. Пользователь обязуется самостоятельно отслеживать изменения в Политике конфиденциальности. В случае внесения существенных изменений, Администрация уведомит Пользователей через официальный Telegram канал или через сам Бот. Продолжение использования Бота после внесения изменений означает согласие Пользователя с обновленной Политикой. Дата последнего обновления всегда указана в заголовке страницы. Заключительные положения
Настоящая Политика конфиденциальности является неотъемлемой частью Пользовательского соглашения и регулирует все аспекты обработки персональных данных в рамках использования Бота MelloANTI.
