Шифрование сообщений

Полное шифрование всех исходящих и входящих сообщений с использованием симметричного алгоритма AES-256 и Fernet. Ключи генерируются индивидуально для каждого пользователя на основе номера телефона и соли, что обеспечивает уникальность и невозможность подбора.

encrypted = Fernet(key).encrypt(text)

Анти-снос сессий

Система непрерывного мониторинга всех активных сессий с проверкой цифровых отпечатков. При обнаружении подозрительной сессии с несовпадающим fingerprint происходит автоматическая блокировка и уведомление пользователя с детальным логированием.

fingerprint = SHA256(dc_id + auth_key + server)

Защита от перехвата

Многоуровневая защита от MITM-атак, сниффинга и перехвата сессий. Используется ротация ключей каждые 24 часа, привязка к устройству через уникальный device_id и проверка IP-адресов с автоматической блокировкой подозрительных.

auto_rotation = 86400 seconds

Блокировка IP

Ручная и автоматическая блокировка IP-адресов при подозрительной активности. Поддерживается постоянная и временная блокировка с возможностью указания причины и срока действия. Все заблокированные IP сохраняются в базе данных.

block_ip(ip, reason, expires_at)

Ротация ключей

Автоматическая и ручная ротация ключей шифрования для обеспечения максимальной безопасности. При ротации старый ключ архивируется в таблице key_rotation с указанием причины и времени, что позволяет отслеживать историю изменений.

rotate_key(user_id, "manual rotation")

Планирование сообщений

Отправка зашифрованных сообщений по расписанию с указанием времени в UNIX формате. Сообщения хранятся в базе данных и отправляются автоматически фоновым процессом, что позволяет планировать коммуникацию заранее.

schedule_message(receiver, time, text)
Система шифрования

MelloANTI использует гибридную систему шифрования на основе двух криптографических библиотек: Fernet (симметричное шифрование) и AES-256 в режиме CFB. Каждый пользователь получает уникальный ключ, генерируемый с помощью PBKDF2HMAC с 200000 итераций и солью длиной 32 байта.

  • Генерация ключа на основе номера телефона
  • Использование соли для уникальности
  • 200000 итераций PBKDF2 для стойкости
  • Автоматическая ротация каждые 24 часа
  • Хранение истории ключей для аудита
  • Поддержка ручной смены ключа

Алгоритм шифрования

# Генерация ключа
salt = os.urandom(32)
kdf = PBKDF2HMAC(algorithm=SHA256, length=32, salt=salt, iterations=200000)
key = base64.urlsafe_b64encode(kdf.derive(phone.encode()))

# Шифрование
f = Fernet(key)
encrypted = f.encrypt(text.encode()).decode()

# Дешифрование
decrypted = f.decrypt(encrypted.encode()).decode()
Управление сессиями

Система управления сессиями обеспечивает полный контроль над всеми активными подключениями к вашему аккаунту. Каждая сессия получает уникальный fingerprint на основе DC ID, ключа авторизации и адреса сервера.

  • Отслеживание всех активных сессий
  • Автоматическая блокировка подозрительных
  • Детальная информация об устройстве и IP
  • Очистка неактивных сессий каждые 24 часа
  • Возможность принудительного завершения

Безопасность и защита

Многофакторная защита

Комбинация шифрования, анти-сноса, ротации ключей и мониторинга сессий создает многоуровневую систему безопасности, которая предотвращает несанкционированный доступ к аккаунту.

Автоматическое резервирование

Ежедневное создание резервных копий всех данных пользователя, включая ключи шифрования, настройки защиты и историю сессий. Резервные копии хранятся локально.

Аудит и логирование

Полное логирование всех действий пользователя с указанием времени, типа действия и IP-адреса. Логи хранятся 30 дней с автоматической очисткой старых записей.

Фильтрация спама

Встроенная система фильтрации нежелательных сообщений на основе регулярных выражений и пользовательских правил. Позволяет автоматически блокировать спам-сообщения.

Защита от брутфорса

Автоматическая блокировка IP-адресов при множественных неудачных попытках авторизации с настраиваемым порогом срабатывания и временем блокировки.

Двухфакторная аутентификация

Поддержка двухфакторной аутентификации с генерацией одноразовых кодов и привязкой к устройству для дополнительного уровня защиты аккаунта.

MelloANTI использует официальный Telegram API через библиотеку Telethon для полной интеграции с аккаунтом. Поддерживаются все основные функции: отправка сообщений, управление сессиями, изменение статуса и многое другое.

Часто задаваемые вопросы

Как активировать бота?

Отправьте команду /start с указанием номера телефона в формате /start +79991234567. После регистрации ключ шифрования будет сгенерирован автоматически.

Как работает шифрование?

Используется симметричное шифрование AES-256 через библиотеку Fernet. Ключ генерируется на основе номера телефона с использованием PBKDF2 и соли.

Что такое анти-снос?

Система мониторинга сессий, которая проверяет цифровые отпечатки всех активных подключений и автоматически блокирует подозрительные сессии.

Как восстановить данные?

Используйте кнопку "Восстановление" и укажите путь к файлу резервной копии. Резервные копии создаются автоматически каждый день.

Безопасно ли хранить ключи?

Ключи хранятся в локальной базе данных SQLite с шифрованием. Доступ к базе данных имеет только владелец аккаунта.

Поддерживается ли рассылка?

Да, поддерживается массовая рассылка зашифрованных сообщений списку получателей с автоматической обработкой ошибок.

Как запланировать сообщение?

Используйте команду "Расписание" и введите ID получателя, время в UNIX формате и текст сообщения.

Что делать при подозрительной сессии?

Бот автоматически заблокирует подозрительную сессию и отправит уведомление. Вы можете вручную очистить все сессии через соответствующую кнопку.

Как обновить ключ шифрования?

Используйте кнопку "Сменить ключ" или "Ротация ключа" для принудительной замены ключа с сохранением истории.

Поддерживается ли двухфакторная аутентификация?

Да, в боте встроена поддержка двухфакторной аутентификации с генерацией одноразовых кодов и привязкой к устройству.
